# 安全说明

## 支持范围

安全修复以最新发布版本为准。较早版本可能需要先升级。

## 报告方式

请不要在公开 Issue 中披露尚未修复的漏洞。

仓库启用 GitHub Security Advisories 后，请从仓库的 **Security** 页面提交私密报告。报告中请包含：

- 受影响的版本
- 问题和可能造成的影响
- 可复现步骤或最小示例
- 建议的修复方式，如有

如果仓库没有显示私密报告入口，请先通过维护者 GitHub 主页公开列出的联系方式请求一个私密沟通渠道；首次联系不要附带漏洞细节、利用代码、令牌或其他敏感信息。

请勿在报告中附带真实令牌、账户密码或其他不必要的敏感数据。

维护者确认问题后，会协调修复、版本发布和公开披露时间。
