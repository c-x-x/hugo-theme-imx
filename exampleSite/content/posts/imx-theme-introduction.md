有些博客急于展示自己：色彩先于文字抵达，动画比段落更早开口。IMX 选择了另一条路。它愿意向后退半步，把屏幕还给文章，把注意力留给那些需要慢慢读完的句子。

![摊开的书页与 IMX 雪花意象](../../images/posts/imx-theme-introduction.png)

这是一款为中文博客而写的 Hugo 主题。它有清楚的内容结构，也有搜索、目录、代码高亮和评论这些现代站点需要的能力；但它并不想把功能一项项摆在读者面前。多数时候，它只是安静地托住内容。

## 一场克制的相遇

打开 IMX，最先感受到的不是某一种强烈风格，而是一种经过整理的秩序。

暖白背景带着纸张的温度，炭黑文字保留足够的分量，棕色与冷灰只在边缘轻轻出现。卡片有阴影，却不悬浮得盛气凌人；玻璃质感藏在导航与控件里，却没有把页面变成一场材质展览。那枚雪花既是标记，也像一处停顿，让理性的技术页面多了一点冬日的清澈。

这种克制并不等于寡淡。恰恰因为装饰被控制在合适的音量，图片、标题与正文才有了各自的位置。读者进入页面后，不必先学会如何使用它，视线自然会落到文章上。

## 阅读始终是第一件事

IMX 对长文格外友好。

正文保持舒展的行宽，中文衬线字体让连续阅读更接近纸面，标题、引用、表格和代码块之间有清楚的节奏。文章足够长时，目录会守在一旁，随着阅读进度标出当前位置；屏幕变窄后，它又收进一个不打扰正文的入口。

代码复制、语言标识和阅读进度都已经准备妥当。它们并不抢眼，却会在真正需要时出现。技术文章因此不必在“好看”和“好用”之间反复取舍：一段代码可以保持准确，一篇散文也不会因此沾上工具页面的生硬。

## 光暗有自己的时间

深色模式常被当作一组相反的颜色，IMX 更愿意把它理解为阅读环境的变化。

白昼里，页面接近温暖的纸色；夜晚到来后，背景沉入柔和的灰黑，文字与边界随之收敛。主题按钮可以在浅色、深色和自动模式之间切换，自动模式依照东八区的时间，在清晨与傍晚完成交接。

Logo、favicon 与 Giscus 评论区会跟随实际主题一起变化。于是深浅色不只是正文背景的替换，而是一套从页面开始、延伸到站点细部的完整状态。

## 每篇文章都有来处

写作久了，真正困难的往往不是发布，而是寻找。

IMX 用文章、分类与标签整理内容。首页保留精选与最新文章，分类呈现一条较长的脉络，标签则容纳那些更细小、更偶然的联系。内置搜索直接读取 Hugo 生成的 JSON 索引，不依赖外部服务，也不要求另建数据库。

这样的结构足够简单，不会让写作者在发布前填写一长串字段；又足够清楚，让几年后的读者仍能从一篇旧文章走到同一主题下的其他文字。归档不是写作之后的杂务，而是内容逐渐形成形状的过程。

## 技术退到幕后

IMX 通过 Hugo Module 安装。站点不需要复制整套主题源码，也不需要为了日常构建安装前端工具链。CSS 由 Hugo Pipes 处理，JavaScript 由 Hugo 构建并生成指纹；普通写作者面对的仍然只是配置文件、Markdown 与图片。

这也让更新变得清楚。主题版本记录在 Go Module 中，需要时升级，遇到问题可以回到先前版本。Logo、头像、默认封面、分享图、菜单和页脚都能从站点配置中替换；更深入的定制，则可以利用 Hugo 的覆盖机制留在自己的仓库里。

技术没有消失，只是被放回了合适的位置。它负责让页面稳定、快速、可以迁移，而不是要求写作者每天照看它。

## 评论可以有，也可以没有

一座博客不一定需要热闹。有人把它当作公开书房，只留下文章；也有人希望读者在文末继续谈话。

因此 IMX 默认关闭评论，但预留了 Giscus。启用后，评论保存在 GitHub Discussions 中，站点本身不需要维护数据库。评论框会懒加载，并跟随页面的深浅色状态。是否打开它，不是主题替你作出的决定，而是写作者对交流方式的选择。

## 它适合怎样的写作者

如果你希望首页像作品集一样不断制造视觉惊喜，IMX 也许过于安静。它并不擅长替内容表演。

但如果你在意中文长文的阅读质感，习惯写技术笔记、项目复盘或缓慢生长的个人文章；如果你希望博客在手机和桌面上都保持从容，也希望几年后仍能用普通 Markdown 带走全部内容，那么 IMX 会是一件耐用的工具。

它不规定博客必须成为哪一种样子。雪花、暖白与深灰只是开端，站点真正的气质最终仍来自作者：来自选择怎样的标题，保留怎样的图片，以及愿意把哪些思考留在公开的时间里。

## 从一篇文章开始

建立博客时，人很容易先沉迷于颜色、域名和部署方式。可一个站点真正开始呼吸，往往是在第一篇文章发布之后。

不妨先写下一段确实想保存的文字，再替换站点名称、头像与封面。等内容站稳以后，再整理分类、开启搜索，或者决定是否需要评论。IMX 的许多配置都允许稍后完成，因为它更关心写作能否顺利开始。

需要动手搭建时，可以继续阅读 [《从空目录到上线：hugo-theme-imx 全量配置手册》]({{% ref "posts/imx-configuration-deployment-guide.md" %}})；若准备开放讨论，再参考 [《把评论留在 GitHub：IMX 主题接入 Giscus》]({{% ref "posts/imx-giscus-comments.md" %}})。

其余的，就交给文章本身。
